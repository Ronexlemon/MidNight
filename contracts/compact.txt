Primitive types:
1. Boolean:-> 
2. Uint<m..n>
3. Uint<n>:-> where n is a non-zero natural number literal or a generic size parameter in scope and bound to a non-zero natural number
4. Field:-> is the type of elements in the scalar prime field of the zero-knowledge proving system
5. [T,...]:-> Where T, ... are zero or more comma-separated types is the type of tuple values with elements types
6. Vector<n,T>, where n is a natural number literal or else a generic size parameteris scope and T is a Type,is a shorthand notation for the tuple type [T,...] with n occurrences of the Type T.
7. Bytes<n>:-> where n is a natural number literal or else a generic size parameter in scope,is the type of byte array values of length n.
8. Opaque<s>:-> Where s is a string literal, is the type of Opaque values with tag s.

User-Defined types:
NB:-> Users can define three kinds of types themselves: structures, enumerations and contracts.

structure types:
    Structure types are defined by a declaration beginning with the keyword struct. 

Enumerations types:
    It introduce a named enumeration type. Each enumeration declaration introduces a distinct type.
    An enumeration declaration has a sequence of named values separated by commas. A trailing separator is allowed but not required


Default Values:
    . Boolean:-> false
    . Uint<0..n> and uint<n>:-> 0
    . Field:-> 0
    . [T, ...] where T, ... is a sequence of zero or more types:-> the tuple with the corresponding length, each of the default value of the corresponding type
    . Bytes<n>:-> the byte array of length n with all zero Bytes
    . Opaque<"string">:-> an empty string. i.e ""
    . Opaque<"Uint8Array">:-> a zero-length Uint8Array,i.e new Uint8Array(0)
    . structure types:->  the struct with all fields set to the default value of their type.
    . enumeration types:-> the first value listed in the declaration.
